## 📁 Google Drive Upload Instructions

Your internship workspace contains a specific set of files representing Day 2. When uploading to your cohort's shared Google Drive, it is best to replicate the structure below to keep your work highly structured and easy to grade.

### 1. Replicated Drive Directory Layout
Create a folder named **`Day 2 - Data Cleaning & SQL Database Design`** in your personal student folder on the Drive, and arrange the files like so:

```text
[Your Personal Student Folder] /
│
└── Day 2 - Data Cleaning & SQL Database Design /
    │
    ├── bluestock_mf.db            # Loaded SQLite Database file (~10.5 MB)
    ├── schema.sql                 # Star Schema DDL Script
    ├── queries.sql                # 10 Analytical SQL Queries Script
    ├── data_dictionary.md         # Database Reference Markdown
    │
    ├── data /
    │   ├── raw /                  # Raw source CSV datasets
    │   └── processed /            # 10 Cleaned CSV files (Day 2 outputs)
    │
    └── scripts /                  # Python ETL and harness files
        ├── data_cleaning.py
        ├── db_loader.py
        └── execute_queries.py
```

### 2. Checklists of Files to Upload

| File Name | Location in Repo | Role / Purpose |
| :--- | :--- | :--- |
| **`bluestock_mf.db`** | `day 2/` | SQLite database file containing fully populated tables. |
| **`schema.sql`** | `day 2/` | The DDL script defining the relational star-schema tables and indexes. |
| **`queries.sql`** | `day 2/` | Standard SQL scripts for the 10 analytical business queries. |
| **`data_dictionary.md`** | `day 2/` | The data dictionary detailing columns, data types, and business definitions. |
| **10 Processed CSVs** | `day 2/data/processed/` | The transformed and forward-filled CSV files generated by the ETL script. |
| **3 Python Scripts** | `day 2/` | `data_cleaning.py`, `db_loader.py`, and `execute_queries.py`. |
